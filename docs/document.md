# Документы (Document)

## Назначение
- **Цель** Обрабатывать файлы и заметки, связанные с оборудованием (пока реализовано только как заготовка).

## Текущее состояние
- **Модель** `application/models/DocumentModel.php` содержит класс-заготовку без методов.
- **Контроллер/Представления** отсутствуют; предполагаемая интеграция — через `EquipmentModel` и карточки заявок.

## Структура данных
- **Таблица `document`** (см. `sql.sql`):
  - `id` — первичный ключ.
  - `equipment_id` — FK на `equipment.id`.
  - `filename` — путь или название файла.
  - Индекс `doc-equip` ускоряет выборку по оборудованию.

## Потенциальные сценарии
- **Загрузка файлов** для оборудования.
- **Хранение технической документации** (PDF, заметки).
- **Привязка к заявкам** через оборудование.

## Рекомендации по реализации
- **Контроллер/CRUD** добавить маршруты для загрузки/удаления.
- **Хранилище** определить директорию хранения и правила именования.
- **UI** интегрировать с `requestEdit.php` или отдельной страницей.
- **Безопасность** проверять типы файлов, размер и права доступа.
